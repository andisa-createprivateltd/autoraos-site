# CreatePrivate Dealer OS (Flutter)

Flutter implementation of CreatePrivate Dealer OS (BETA), aligned to the existing Next.js/Supabase backend and dealer-scoped RLS model.

## What is included

- Supabase Auth email/password login + password reset
- Dealer/role-scoped modules:
  - Login
  - Dashboard (Money View)
  - Conversations (inbox + thread)
  - Leads (filters + detail)
  - Bookings (list + create + status update)
  - Settings (admin role)
  - Analytics (trend + source split)
- Role permissions for:
  - `dealer_admin`
  - `dealer_sales`
  - `dealer_marketing`
  - `platform_owner`
  - `platform_support`
- Message privacy controls for marketing via feature flag from dealer settings
- Phone masking by default with admin override support
- Offline cache (SQLite):
  - Last 50 conversations per dealer
  - Last 50 messages per conversation per dealer
- Push plumbing via Firebase Messaging (FCM/APNs on iOS)
- Mock mode for backend-free demo

## 2) Run (mock mode)

No backend required:

```bash
flutter run \
  --dart-define=ENABLE_MOCK_MODE=1 \
  --dart-define=ENABLE_PUSH=0
```

## 3) Run (live Supabase/backend)

Pass values with `--dart-define`:

```bash
flutter run \
  --dart-define=SUPABASE_URL=https://YOUR_PROJECT.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY \
  --dart-define=BACKEND_BASE_URL=https://YOUR_BACKEND_DOMAIN \
  --dart-define=ENABLE_MOCK_MODE=0 \
  --dart-define=ALLOW_ADMIN_PHONE_UNMASK=1 \
  --dart-define=MARKETING_CAN_VIEW_MESSAGES_DEFAULT=0 \
  --dart-define=MISSED_LEAD_MINUTES=10 \
  --dart-define=ENABLE_PUSH=1
```

## 4) iOS signing and device run

1. Open iOS module once generated by Flutter:
   - `open ios/Runner.xcworkspace`
2. In Xcode, set Team and a unique bundle identifier.
3. Enable `Push Notifications` and `Background Modes > Remote notifications`.
4. Build/run from Flutter or Xcode.

## 5) Push setup (FCM + APNs)

1. Create Firebase project and add iOS app.
2. Download `GoogleService-Info.plist` and add it to `ios/Runner/`.
3. In Apple Developer portal, enable APNs for the bundle ID.
4. Upload APNs auth key in Firebase Cloud Messaging settings.
5. Backend push payload for deep links must include:
   - `type`: `conversation` or `booking`
   - `id`: record UUID
6. App registers token and posts to `POST /api/device-token`.

## 6) Security notes

- WhatsApp sends are backend-only (`/api/send-message`).
- No Meta Cloud API token is stored in mobile app.
- Dealer data isolation depends on Supabase RLS policies.
- Session tokens are managed by Supabase + secure storage.
- UI masks phones by default.
