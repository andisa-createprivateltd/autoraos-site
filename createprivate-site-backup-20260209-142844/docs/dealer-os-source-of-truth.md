# CreatePrivate Dealer OS - Source Of Truth

## 1. Dealer OS V1 UI Structure

### Objective
Dealer OS is a revenue control system for dealerships, not a visual dashboard product.

### V1 Information Architecture

1. Login
- platform_owner
- platform_support
- dealer_admin
- dealer_sales
- dealer_marketing (metrics only)

2. Dashboard (Money View)
- New Leads (last 24h)
- Response Time (avg)
- Bookings Created
- Missed Leads Alert
- Conversion Snapshot (basic)

3. Conversations (Core)
- WhatsApp-style inbox
- One thread per lead
- AI + human messages inline
- Tags: New / Hot / Booked / Visited / Sold / Lost
- Lead source label: Ads / Website / OEM

4. Bookings
- Test drives, appointments, call-backs
- Calendar view
- Auto confirmations
- No-show reminders
- Status: Booked / Completed / No-show

5. AI Assistant (Admin)
- Business hours
- FAQs (models, pricing range, finance)
- Booking rules
- Escalation logic for AI handoff

6. Insights (Minimal)
- Avg response time
- Leads vs bookings
- Missed after-hours leads

7. Settings
- Users
- WhatsApp numbers
- Booking availability
- Billing plan

### Not In V1
- White-labelling
- Multi-language
- Advanced analytics
- Custom workflows

## 3. Founder Narrative

"CreatePrivate builds the WhatsApp and lead operating system for automotive dealerships.

Dealerships lose revenue not because of lack of demand, but because leads are slow, fragmented, and poorly handled, especially on WhatsApp.

We sit directly in the dealer's communication flow, responding instantly, qualifying leads, booking test drives, and tracking conversion in real time.
